User Guide

Overview

ZPR Policy Builder is a browser-based editor for writing and testing ZPL (Zero-trust Policy Language) policies. ZPL provides a structured, human-readable way to express access control rules.

  • Write ZPL incrementally — enter new statements, check them, accept or discard
  • Organize policies into a namespace tree with delegated ownership
  • Generate and run test suites including AI-generated adversarial tests
  • Run the AI-powered Policy Audit to detect shadowing, gaps, and overly broad rules
  • Save audit results as versioned Reports

Policy Editor

The Policy Editor has two zones. The upper zone is where you enter new ZPL. Type one or more statements, click Check ZPL, and any errors appear directly below your text. If valid, an Accept / Discard / Send to AI bar appears. Clicking Accept adds the new statements to the full policy shown in the lower panel.

The lower zone shows all accepted policies for the active namespace. Switch between Table view (with Edit / Delete / yaml per row) and ZPL view (the full canonical text).

Namespace Selector

The namespace tree in the left nav shows your full hierarchy. Click any namespace to make it active — all pages operate on the active namespace.

Policy Audit

Run a Fast audit (a few seconds, Haiku model) or a Deep audit (~2 minutes, Sonnet model) on the active namespace and all sub-namespaces. Results show color-coded issues (HIGH / MEDIUM / LOW) with findings, recommended fixes, and one-click + Add buttons for fixes expressible as new ZPL rules.

Reports

After an audit completes, click Save to store the result as a versioned report. Reports are listed in the Reports page, newest first.
