"Creating a ZPR Policy\nCreate a Zero Trust Packet Routing (ZPR) policy in the Zero Trust Packet Routing service.\n\nFamiliarize yourself with ZPR policy basics before trying to create a policy.\n Note\nA ZPR policy can be created only in the root compartment of a tenancy.\n\nConsoleCLIAPI\n\nOn the Policies list page, select Create policy. If you need help finding the list page, see Listing ZPR Policies.\n\nEnter a name and a description for the policy.\n\nSelect Add policy statements.\n\nSelect which type of policy builder you want to use to create the policy statements:\n\nPolicy template builder\n lets you select from a list of templates based on common use case scenarios that provide prefilled ZPR policy statements that you can then customize to create a ZPR policy. \n\nSimple policy builder lets you select from pre-populated lists of resources identified by their security attributes to express security intent between two endpoints. The policy builder automatically generates the policy statement using correct syntax.\n\nManual policy builder lets you enter free-form policy.\n\n Note If you change to a different type of policy builder while creating a policy, then ZPR resets the policy statements. \n\nWhen you're finished adding policy statements, select Add.\n(Optional) \n                            To apply tags to the policy, do one of the following depending on what you see: \n\nExpand the Tags section, select Add tag, and then enter the required values.\nSelect Show advanced options and enter the required values.\n\nIf you have permissions to create a resource, you also have permissions to apply free-form tags to that resource. To apply a defined tag, you must have permissions to use the tag namespace. For more information about tagging, see Resource Tags. If you're not sure whether to apply tags, skip this option or ask an administrator. You can apply tags later.\n\nSelect Create policy.\n\nThe ZPR policy is enabled by applying security attributes to resources.\n\nUse the oci zpr zpr-policy create command and required parameters to create a Zero Trust Packet Routing (ZPR) policy:\nCommandCopyTry Itoci zpr zpr-policy create --compartment-id <compartment_ocid> --description <zpr_policy_description> --name <zpr_policy_name> --statements <zpr_policy_statements> [OPTIONS]\n\nFor a complete list of parameters and values for CLI commands, see the CLI Command Reference.\n\nRun the CreateZprPolicy operation to create a Zero Trust Packet Routing policy."
