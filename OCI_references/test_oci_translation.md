## VCN Context → ZPL Namespace Mapping

OCI `in <attr> VCN` clauses scope a rule to a specific virtual network. ZPL has no VCN concept. The table below maps each VCN attribute to a suggested ZPL namespace — create a child namespace with that name and place the corresponding rules inside it.

| OCI VCN Attribute | Suggested ZPL Namespace |
|---|---|
| `app:fin-network` | `fin-network` |
| `applications.apps:app1` | `app1` |
| `front-end:network` | `network` |
| `VCN-Network:DB` | `DB` |
| `networks:net1` | `net1` |
| `finance.network:prod` | `prod` |
| `applications.vcn:A` | `A` |
| `database.vcn:B` | `B` |
| `networks:net2` | `net2` |
| `finance.network:dev` | `dev` |
| `VCN-Network:App` | `App` |
| `VCN-Network:Remote` | `Remote` |
| `networks:application` | `application` |
| `networks:internal` | `internal` |
| `my:VCN` | `VCN` |
| `vcn:A` | `A` |
